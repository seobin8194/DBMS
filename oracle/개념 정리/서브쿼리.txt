서브 쿼리
sql 안에 sql이 있는 형태

FROM절에 SQL문 쓰는거 : INLINE VIEW
-> 가상 테이블에서 가져오고 싶을때
SELECT절에 SQL문 쓰는거 : SCALAR
-> 같이 검색할 수 없는 경우(일반 컬럼과 그룹함수)
WHERE절에 SQL문 쓰는거 : SUB QUERY
-> WHERE절에서 그룹함수를 사용할 수 없다

전체 틀을 작성하고 서브쿼리 작성하고 전체 틀의 서브쿼리자리에 복사 붙여 넣기


뷰
가상의 테이블
변수에 담아 놓는것과 비슷

뷰를 생성하기 위해서는 CREATE VIEW라는 권한이 필요
CMD에서 SYSTEM 계정으로 로그인해서 권한 주기
GRANT CREATE VIEW TO 계정명;

뷰 생성
서브쿼리 결과대로 가상 테이블 만들기
CREATE VIEW 뷰이름(컬럼명1, 컬럼명2,,,) AS (서브쿼리)

뷰삭제
DROP VIEW 뷰이름


ANY(서브쿼리) : 서브쿼리 결과의 단 한행이라도 조건식을 만족하면 참
ALL(서브쿼리) : 서브쿼리 결과의 모든 행이 조건식을 만족하면 참


EXISTS 서브쿼리 : 서브쿼리의 결과가 존재하기만 하면 참


UNION
두 쿼리문의 결과를 합칠때 사용
각 결과 테이블을 위아래로 합친다(행 추가) -> 두 쿼리문의 컬럼개수가 동일해야 가능
중복된 데이터들은 배제된다
UNION ALL
중복제거하지 않고 합친다
