무결성
데이터의 정확성 일관성 유효성이 유지되는 것
정확성 : 데이터는 애매하지 않아야 한다
일관성 : 각 사용자가 일관된 데이터를 볼 수 있도록 해야한다
유효성 : 데이터가 실제 존재하는 데이터야한다

1. 개체 무결성
모든 테이블이 pk로 설정된 컬럼을 가지고 있어야한다
pk로 설정된 컬럼은 고유한 값을 가져야 하고 반드시 null값이 아니여야한다
2. 참조 무결성
동일한 데이터를 갖는 두 테이블의 데이터가 항상 일관된 값을 갖도록 유지
-> 테이블의 데이터를 수정할때 업데이트할 곳을 다 찾아서 수정해야한다
3. 도메인 무결성
컬럼의 타입, null값 허용 등에 대한 사항을 정의하고 올바른 데이터가 입력되었는지 확인

모델링
추상적인 주제를 db에 맞게 설계하는 것
1. 요구분석
영화관을 만들기 위해 필요한 것들 나열
-> 영화 상영관 시간표
2. 개념적 설계
각 테이블에 어떤걸 넣고 테이블끼리 관계는 어떤지 개념으로만 생각
영화      >>      시간표      <<       상영관
제목               상영시간            상영관 이름
감독              상영관이름            수용인원
상영시간          영화제목
3. 논리적 설계
영화      >>         시간표      <<       상영관
영화코드(pk)      상영시간(pk)        상영관 이름(pk)
제목               상영관이름(fk)           수용인원
상영시간          영화제목(pk)
감독                 영화코드(fk)

시간표에 들어가는 속성은 영화와 상영관 테이블에 있으므로 fk를 통해서 가져온다
4. 물리적 설계
메모리에 넣을 준비 -> 자료형을 정함
영화             >>         시간표        <<         상영관
영화코드(pk)(vc2)      상영시간(pk)(date)    상영관 이름(pk)(vc2)
제목(vc2)               상영관이름(fk)(vc2)         수용인원(num)
상영시간(date)          영화제목(pk)(vc2)
감독(vc2)                 영화코드(fk)(vc2)
